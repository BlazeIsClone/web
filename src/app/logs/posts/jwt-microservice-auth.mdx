---
title: "Authenticate API Microservice with JWTs"
publishedAt: "2026-04-11"
summary: "Use JWTs to authenticate separate API Microservice"
---

To handle user data we can implement an Identity Provider Service (IDP) which is responsible for signing JWTs with asymmetrical keys (RS256) and storing user information.
The Identity Provider also has an open endpoint which exposes the public key in the form of a JWK (JSON Web Key), which is used to validate issued tokens by any other service. Ideally the external service would cache the JWK to reduce traffic to the IDP.

This also poses another issue, we will have to implement more code to validate the tokens with the JWK endpoint. This is where an API Gateway comes in. The API Gateway sits between the frontend client and the API server, acting as a checkpoint.
The API Gateway caches the JWK using the IDP endpoint and validates all incoming requests.
This means we only have to implement features like JWK validation, rate-limiting, and SSL on the API Gateway and will not have to rely on internal services for these.

Another improvement to the API Gateway would be to write the decoded JWT data onto the request headers, so the gateway can pass decoded data such as `x-jwt-email: person@email.com` directly to the internal services.
